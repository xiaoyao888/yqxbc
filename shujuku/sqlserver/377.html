检测SQL Server 2000是否优化
yqxbc 发布时间：04-01 来源：一起学编程 浏览：14次

文章目录

        问：我的服务器是x366 (02 CPU 3.16GB, 4GB 内存, 6x73GB HDD/RAID5-C, D, E, F, CD-ROM)，操作系统是微软的Windows 2003企业版，用作数据库服务器(C/S 结构应用程序)的是SQL Server 2000标准版(SP4)。我有两个问题想请教您: 

        1、这里是我的SQL Server 2000: 

        C: (操作系统 + SQL程序 ) 

        D: (50GB 用于数据库数据文件 .MDF) 

        E: (50GB 用于数据库日志文件.LDF) 

        F: (其他数据) 

        你认为这种情况是经过优化的吗? 

        2、这是我对SQL 2000 Enterprise最大支持2GB内存的理解。我能否在RAM中使用TEMPDB来加强性能?如果这样的话，那么我该如何配置我的系统来获得优化的性能? 

        答：1、将不同的处理类型分开是个很好的配置。在没有执行任何容量计划的时候，很难判断这个存储是否充分。此外，基于控制器的配置，用独立的物理磁盘要比一个RAID 5集合更好。评估一下在测试环境中服务器的性能，看看结果如何。如果问题是磁盘驱动(队列)相关的问题，那么考虑增加额外的磁盘和RAID集合来支持负载。 

        2、与Windows 2003一同使用的SQL Server 2000版可以支持高达2GB的内存。如果你升级到SQL Server 2000企业版，那么你就可以使用总量为32GB的内存。 

        RAM中的TempDB 是SQL Server 6.5的一个选项，在SQL Server 2000 中不复存在了。
